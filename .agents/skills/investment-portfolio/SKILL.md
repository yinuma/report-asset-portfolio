# Investment Portfolio Analyzer — マルコフモデル設計

このスキルはマルコフモデルの考え方に基づいています。

```
S(N) = f( S(N-1), POLICY, O(N) )

S(N-1) : 前日のポートフォリオ状態 → CURRENT.md
POLICY  : 投資方針（低頻度変更）   → POLICY.md
O(N)    : 当日の市場観測           → Web検索で取得
A(N)    : 提案される行動           → リバランス提案レポート
S(N)    : ユーザー確定後の新状態   → CURRENT.md を上書き更新
```

次の状態は「直前の状態 + 方針 + 最新の観測」だけで決まります。過去の全履歴を遡る必要はなく、CURRENT.md と POLICY.md の2ファイルが今の意思決定に必要な十分な情報を持っています（変更履歴は参考として CURRENT.md 内に保持します）。

過去のレポートは `reports/` ディレクトリに日時付きファイル名で蓄積されます。意思決定には不要ですが、振り返りや傾向分析に使えます。

**重要な前提**: あなたは投資助言業の登録を受けたファイナンシャルアドバイザーではありません。すべての出力は情報提供・分析目的であり、投資判断の最終決定はユーザー自身が行うものです。レポートの冒頭と末尾にこの旨を必ず明記してください。

## ディレクトリ構成

```
investment-portfolio/
├── SKILL.md          ← このファイル
├── POLICY.md         ← 投資方針（遷移ルール）
├── CURRENT.md        ← ポートフォリオ状態 S(N-1)
└── reports/          ← 過去のレポートを日時付きで蓄積
    ├── report-2026-03-10-090000.md
    ├── report-2026-03-13-093000.md
    └── ...
```

## データファイル

### POLICY.md — 投資方針（遷移ルール）

投資方針は頻繁に変わるものではないため、独立したファイルで管理します。日々の分析ではこのファイルを読み込み、すべての提案がこの方針に沿っているかを確認してください。

**重要ルール（更新）**:
- POLICY.md では、**中長期（3か月〜数年）**の投資方針のみを記述すること。
- 超短期（当日〜1週間以内）の売買方針は POLICY.md に書かず、**reports 側に集約**すること。
- POLICY.md の対象資産・候補は、**MONEX証券で購入可能な具体銘柄**（米国株・日本株・ETF・債券）で明示すること。

```markdown
# 投資方針（Investment Policy）

**策定日**: YYYY-MM-DD
**最終見直し日**: YYYY-MM-DD

## 基本方針
- **投資目的**: （例）老後資金形成 / 資産形成 / インカム重視 etc.
- **投資期間**: 長期（10年以上）/ 中期（3-10年）/ 短期（1-3年）
- **リスク許容度**: 積極的 / 中程度 / 保守的
  - 許容最大ドローダウン: -XX%
  - 年間目標リターン: X-X%（参考値）

## 目標アセットアロケーション
| 資産クラス | 目標比率 | 許容レンジ | 備考 |
|-----------|---------|-----------|------|
| 米国株 | XX% | ±X% | |
| 日本株 | XX% | ±X% | |
| 先進国株（米国除く）| XX% | ±X% | |
| 新興国株 | XX% | ±X% | |
| 債券 | XX% | ±X% | |
| 外貨 | XX% | ±X% | |
| コモディティ | XX% | ±X% | |
| 現金 | XX% | ±X% | 最低X%を維持 |

## 採用ユニバース（MONEXで購入可能な具体銘柄）
- 米国株/ETF: 例 `VTI`, `VOO`, `SPY`, `XLE`, `GLD`, `TLT`, `SHY`
- 日本株/ETF: 例 `7203`, `6758`, `8306`, `1306`, `1489`
- 債券/債券ETF: 例 `AGG`, `BND`, `IEF`, `SHY`

※実際の POLICY では、抽象カテゴリだけでなく採用候補銘柄を具体的に列挙すること。

## リバランスルール
- **トリガー条件**: 目標比率から±X%以上乖離した場合
- **リバランス頻度**: 月次 / 四半期 / 乖離時のみ
- **方法**: 一括 / 段階的（X回に分けて）

## 制約条件
- **セクター制限**: （例）テック偏重を避ける。単一セクター上限X% etc.
- **個別銘柄上限**: ポートフォリオの X% 以下
- **ESG方針**: 考慮する / しない
- **除外対象**: （例）タバコ、兵器 etc.
- **税務考慮**: NISA枠優先 / 損益通算意識 etc.

## 参考情報
- **メイン証券口座**: （例）SBI証券
- **積立設定**: 毎月X万円
- **次回方針見直し予定**: YYYY-MM-DD
```

POLICY.md が存在しない場合、ユーザーに投資方針をヒアリングし、新規作成してください。最低限「目標アロケーション」と「リスク許容度」があれば分析は開始できます。

ユーザーが方針変更を伝えてきた場合は POLICY.md を更新してください。

### CURRENT.md — ポートフォリオ状態 S(N-1)

前日時点（直近の確定状態）のポートフォリオです。日々の分析のたびにこのファイルを起点とします。

```markdown
# ポートフォリオ状態

**状態日付**: YYYY-MM-DD
**総資産評価額**: ¥XX,XXX,XXX（状態日付時点の概算）

### 日本株

### 外貨

|------|------|------|
```

CURRENT.md が存在しない場合、ユーザーにポートフォリオの情報を聞き取り、新規作成してください。

## ワークフロー

### Phase 1: 状態読み込み — S(N-1) + POLICY

1. スキルディレクトリ内の **POLICY.md** を読み込む → 投資方針を把握
2. スキルディレクトリ内の **CURRENT.md** を読み込む → 現在のポートフォリオ状態を把握
3. いずれかが存在しない場合はユーザーにヒアリングし新規作成する
4. ユーザーから追加情報（追加資金、方針変更、特定の懸念など）があればメモする

### Phase 2: 観測取得 — O(N)

WebSearchを使って当日の市場データを収集します。検索クエリには必ず本日の日付を含めてください。

### Emergency Monitor Mode（30分監視）

定期日次レポートとは別に、30分おきの監視実行では以下の判定を行います。

1. 主要トリガーを確認
   - 原油（Brent/WTI）の急変（目安: 日次 ±5%以上）
   - USDJPY の急変（目安: 日次 ±2%以上）
   - 金利急変（米10年債など）
   - 地政学ヘッドライン（海峡封鎖・制裁拡大・軍事衝突拡大）
2. トリガー判定
   - 1つ以上該当 → **緊急局面**
   - 該当なし → **平常**
3. 出力ルール
   - 緊急局面: レポート作成・要約通知を実施
   - 平常: **レポート作成しない / CURRENT.md更新しない / Git更新しない / 通知しない**

平常時は明示的に `NO_REPLY` を返し、更新をスキップしてください。

**収集対象（優先度順）**:

1. **保有銘柄の株価**（最優先）
   - CURRENT.md に記載された各銘柄の現在値と騰落率
   - `"{ticker} stock price today {date}"`
   - `"{銘柄名} 株価 {date}"`

2. **主要インデックス・為替・金利**
   - S&P 500, NASDAQ, 日経225, TOPIX
   - USD/JPY, EUR/JPY 等の為替レート
   - 米国10年債、日本国債の利回り
   - `"S&P 500 today {date}"`, `"USD JPY {date}"`

3. **マクロ経済ニュース**
   - FRB・日銀・ECBの金融政策
   - 主要経済指標（CPI、雇用統計、GDP等）の最新発表
   - 地政学リスク
   - `"market news today {date}"`, `"経済ニュース 今日 {date}"`

4. **ポートフォリオ関連のセクター動向**
   - 保有銘柄のセクターに関するニュース

ポートフォリオに含まれない資産クラスの詳細調査は省略して構いません。

### Phase 3: 分析 — f(S, POLICY, O)

収集したデータと POLICY.md の方針を照合しながら分析します。

**3a. 時価評価の推定**
- CURRENT.md の保有数量 × 現在の株価で各ポジションの時価を推定
- 資産クラス別の現在比率を算出

**3b. 乖離チェック**
- POLICY.md の目標アロケーションとの乖離幅を計算
- POLICY.md のリバランストリガー条件（±X%）に該当するか判定
- 個別銘柄上限、セクター制限等の制約条件への抵触チェック

**3c. 市場環境評価**
- 現在の市場環境がポートフォリオにとって有利か不利か
- 短期的なリスク要因（1-3ヶ月）
- ポートフォリオに影響しうる今後のイベント

**3d. リバランス緊急度の判定**

POLICY.md の許容レンジに基づいて判定:
- 🔴 **即時対応推奨**: 許容レンジを超える乖離 or 重大なリスク変化
- 🟡 **次回定期チェック時に対応**: 乖離はあるがレンジ内 or 軽微
- 🟢 **現状維持**: 乖離は小さく、市場環境もポリシーと整合的

### Phase 4: 行動提案 — A(N)

POLICY.md の方針に沿った具体的な提案を作成します。

**重要ルール（更新）**:
- 提案する銘柄・ETF・債券は、**MONEX証券で購入可能な具体銘柄**で記載すること（抽象カテゴリのみの記述は禁止）。
- reports では、POLICY と当日までの市場動向を踏まえ、**超短期（当日〜1週間以内）の推奨行動**を必ず提示すること。
- 超短期提案は、**優先度（High / Medium / Low）付き**で並べること。

**複数プラン提示**:
- **プランA（POLICY準拠）**: 方針通りの機械的リバランス。目標比率に最も近づける売買。
- **プランB（市場環境考慮）**: 当日の市場環境を踏まえたタクティカルな調整。ただしPOLICY.mdの許容レンジ内で。

各プランについて:
- 具体的な売買候補（銘柄、数量の目安、概算金額）
- 追加資金がある場合の配分提案
- 実行時の注意点（税務影響、取引コスト、POLICY.mdの制約との整合性）
- POLICY.md のリバランスルール（一括/段階的）に従った実行スケジュール

### Phase 5: レポート出力

レポートはスキルディレクトリ内の `reports/` ディレクトリに保存します。ディレクトリが存在しない場合は作成してください。

**ファイル名規則**: `report-YYYY-MM-DD-HHMMSS.md`（例: `report-2026-03-13-093000.md`）

レポートを保存したら、そのファイルパスをユーザーに伝えてください。過去のレポートは `reports/` に蓄積されていくので、ユーザーが後から振り返ることができます。

### Phase 6: 状態遷移 — S(N-1) → S(N)

レポート提出後、ユーザーが最終的なリバランス結果を伝えてきたら:

1. ユーザーの報告内容を確認する（「プランAで実行した」「VTIを5株追加した」等）
2. **CURRENT.md を更新する**:
   - 状態日付を本日に更新
   - 保有資産の数量・銘柄を実際の売買に基づいて更新
   - 総資産評価額を推定更新
   - 変更履歴に今回の変更を追記
3. 更新後の CURRENT.md の内容サマリーをユーザーに提示し、確認を取る

ユーザーが「提案通りにした」と言った場合は、該当プランの内容に基づいて更新してください。数量が曖昧な場合はユーザーに確認してください。

ユーザーが「何もしなかった」「見送り」と言った場合は、CURRENT.md は更新不要です。次回のチェック時にまた同じ S(N-1) から出発します。

## 投資方針変更への対応

ユーザーが投資方針の変更を伝えてきた場合（「リスク許容度を下げたい」「目標配分を変えたい」等）:

1. 変更内容を確認する
2. **POLICY.md を更新する**（最終見直し日も更新）
3. 変更前後の比較を提示する
4. 必要に応じて、新しい方針に基づいたリバランス提案を行う

POLICY.md の変更は遷移ルールの変更に相当するため、その影響範囲を説明してください。

## 注意事項

- **データの鮮度を明示する**: 株価がいつ時点のものか必ず記載。Web検索結果はリアルタイムではない場合があります。
- **不確実性に正直であること**: 断定的な表現は避け、「〜の可能性がある」「〜と見られている」のような表現を使ってください。
- **情報の限界**: 取得できなかったデータは正直にその旨を記載し、推定値を使う場合は明記してください。
- **POLICY.md への忠実さ**: すべての提案は POLICY.md の方針に沿ったものにしてください。方針から逸脱する提案をする場合は、その理由を明確に説明し、方針の見直し自体を提案してください。
- **簡潔さ**: サマリーで全体像を伝え、詳細は各セクションで展開する構成にしてください。
